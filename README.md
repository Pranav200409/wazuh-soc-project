# Wazuh SOC Project

## 📌 Overview
This project demonstrates the implementation of a Security Operations Center (SOC) monitoring environment using Wazuh, an open-source security platform for threat detection, log analysis, file integrity monitoring, and incident response.

The objective of this project is to simulate real-world cyber attacks and analyze how a SOC analyst can detect, investigate, and respond to these threats using Wazuh and its integrated monitoring capabilities.

The project includes attack simulations, alert generation, and dashboard visualization to understand how security events are monitored and managed in a SOC environment.

## 🎯 Objectives
The main objectives of this project are:
- To deploy and configure a Wazuh SIEM environment
- To monitor system activities and detect suspicious behavior
- To simulate common cyber attacks in a controlled environment
- To analyze alerts generated by Wazuh
- To understand how SOC analysts investigate security incidents
- To visualize security events through dashboards

## 🏗️ SOC Architecture
The SOC environment in this project consists of multiple components that work together to monitor and analyze security events.

Components
1. Wazuh Manager
Centralized server responsible for collecting and analyzing logs
Applies security rules to detect suspicious activity

2. Wazuh Agent
Installed on monitored endpoints
Collects system logs and sends them to the Wazuh Manager

3. Endpoint System
Ubuntu server acting as the monitored machine
Generates logs during attack simulations

4. Attacker Machine
Kali Linux used to simulate cyber attacks such as SSH brute-force

## Workflow
Attacker (Kali Linux)
        ↓
Target Machine (Ubuntu Server)
        ↓
Wazuh Agent collects logs
        ↓
Logs sent to Wazuh Manager
        ↓
Security rules triggered
        ↓
Alerts generated and visualized in dashboard

## 🔧 Key Features Implemented
1️⃣ File Integrity Monitoring (FIM)
Wazuh monitors critical files and directories for unauthorized changes.
Capabilities:
- Detect file modifications
- Detect file deletions
- Detect new file creation
- Generate alerts for suspicious changes

This feature helps detect:
- Malware activity
- Unauthorized configuration changes
- Insider threats

2️⃣ SSH Brute Force Detection
This project simulates a brute-force login attack against an SSH server.
Wazuh detects:
- Multiple failed login attempts
- Suspicious authentication activity
- Possible brute-force attacks

Security alerts are generated when login attempts exceed defined thresholds.

3️⃣ Threat Simulation (Ransomware-Like Behavior)
A ransomware-like scenario was simulated to observe how Wazuh detects suspicious file activity.
Simulated actions included:
- File deletion
- File modification
- Creation of suspicious files

Wazuh File Integrity Monitoring detected these events and generated alerts.

4️⃣ MITRE ATT&CK Mapping
Detected activities were mapped to the MITRE ATT&CK framework, which is widely used by cybersecurity professionals to classify adversary behavior.
Example mappings:
- Attack	MITRE Technique
- SSH Brute Force	T1110 – Brute Force
- File Modification	T1565 – Data Manipulation
- Unauthorized Access	T1078 – Valid Accounts

This helps SOC analysts understand the tactics and techniques used by attackers.
